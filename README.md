# RAG Assistant (Flask)

Готовый RAG-ассистент на Flask для работы с внутренними документами: загрузка файлов, индексация во векторную базу и ответы на вопросы на основе найденных фрагментов.

## Возможности

- Загрузка и обработка документов через админку
- Разбиение текста на чанки и сохранение в векторное хранилище
- Семантический поиск по embeddings
- Реранжирование найденных фрагментов
- Генерация ответа с опорой на контекст
- Кэширование повторных запросов
- Панель с диагностикой retrieval и generation

## Стек

- Python 3.x
- Flask
- ChromaDB (или другое векторное хранилище)
- Yandex Cloud LLM + embeddings (для генерации ответов и семантического поиска)
- HTML + Jinja2 шаблоны
- Bootstrap / кастомный CSS

## Как запустить локально

### 1. Клонировать репозиторий

```bash
git clone https://github.com/shkapao28-ship-it/rag-assistant.git
cd rag-assistant
```

### 2. Создать виртуальное окружение и установить зависимости

```bash
python -m venv .venv
source .venv/bin/activate    # Linux / macOS
# или
.venv\Scripts\activate       # Windows

pip install -r requirements.txt
```

### 3. Настроить переменные окружения

Создайте файл `.env` в корне проекта (он уже добавлен в `.gitignore`, в репозиторий не попадает) и укажите там настройки, например:

```env
# Доступ к Yandex Cloud
YANDEX_API_KEY=ваш_API_ключ_Яндекса
YANDEX_FOLDER_ID=ваш_folder_id

# Модели для RAG (имена зависят от вашей конфигурации в yandex_client.py)
RAG_EMBEDDINGS_MODEL=text-search-doc
RAG_COMPLETION_MODEL=gpt-lite

# Путь к векторной базе
RAG_VECTOR_DB_PATH=./chroma_data

# Настройки Flask
FLASK_ENV=development
FLASK_PORT=5000
```

Точный набор переменных зависит от реализации `config.py` и `yandex_client.py`, но ключевые параметры — API‑ключ и folder_id Яндекс Cloud, имена моделей для embeddings/генерации и путь к векторному хранилищу.

### 4. Запустить приложение

Из корня проекта:

```bash
python -m rag_service
```

По умолчанию приложение поднимается на `http://127.0.0.1:5000/` (порт можно изменить через переменную `FLASK_PORT` или настройки в `config.py`).

## Как пользоваться

### Основная панель (главная страница)

На главной панели можно:

- Ввести вопрос к базе знаний.
- Выбрать коллекцию/режим поиска (например, `precise`, `balanced`, `recall`).
- Задать `top_k` и другие параметры поиска.

После запроса отображаются:

- Сгенерированный ответ.
- Список найденных фрагментов исходных документов.
- Технические параметры запроса:
  - `top_k` и фактический `fetch_k_used`;
  - количество сырых результатов и отфильтрованных после реранжирования;
  - количество чанков, попавших в контекст;
  - флаг `cache_hit` (был ли ответ взят из кэша);
  - вспомогательные debug‑поля для анализа качества retrieval.

Это позволяет не только видеть ответ, но и понимать, за счёт каких фрагментов и параметров он получился.

### Админка (`/admin`)

В админ‑панели доступны:

- Загрузка новых документов.
- Запуск индексации.
- Очистка кэша.
- Просмотр статистики по базе.

Отображаются:

- Текущая версия индекса.
- Количество файлов во входной папке.
- Количество обработанных файлов.
- Количество документов и чанков в базе.
- Размерность embeddings.
- История запросов (для отладки качества RAG).

## Зачем этот проект

Проект показывает полный цикл Retrieval-Augmented Generation:

1. Загрузка исходных документов.
2. Разбиение на чанки и подготовка текста.
3. Построение embeddings и сохранение во векторную базу.
4. Поиск по векторной базе и реранжирование результатов.
5. Сборка контекста и генерация ответа.
6. Кэширование и диагностика запросов.

Эту структуру можно использовать как основу для реальных внутренних ассистентов (FAQ, база знаний компании, обучение сотрудников и т.п.).
